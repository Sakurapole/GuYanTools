# 多设备剪贴板需求

## 产品定位

多设备剪贴板不是单纯的剪贴板历史窗口，而是面向多设备工作流的剪贴板替代能力。它需要在保留系统剪贴板基本体验的同时，增加跨设备同步、来源设备标识、历史检索和未来可扩展同步后端。

## 首版范围

- 桌面端优先，实现 Electron 端完整可用闭环。
- 基于 mDNS/DNS-SD 做局域网设备发现。
- 首次设备交互必须经过配对确认；未配对设备不得写入本机剪贴板。
- `Alt+V` 唤出右下角窗口，窗口紧靠任务栏上方。
- 窗口聚焦时双击 `Esc` 关闭。
- 每条历史记录显示来源设备标签。
- Android/Flutter 只验证架构可用路径，不做完整移动端实现。

## 内容支持

- 文本：保真保存和同步，展示层识别普通文本、URL、Markdown、emoji 文本等标签。
- 图片：桌面端以 PNG 资产保存，历史窗口展示缩略图。
- 文件：支持文件项历史与小文件同步；最大同步大小可配置，硬上限 1GB。
- 视频：首版按文件项处理，支持常见视频扩展名标签，不做转码。
- 超限或平台无法传输内容：保留本机历史，并标记为本机可用。

## 同步方案取舍

- mDNS 局域网同步：首版实现。优点是无云账号、低延迟、适合多桌面设备；缺点是跨网段、校园网、部分公司网络可能不可用。
- 坚果云 WebDAV：暂缓。适合作为 v2 跨网络/离线同步后端，但需要账号配置、冲突解决、隐私和大文件策略。
- 主同步设备：暂缓。适合作为未来中继/家庭服务器模式，但首版会增加主设备离线、拓扑管理和回放一致性复杂度。

## 配置需求

- `features.multiDeviceClipboard.enabled`：启用/停用多设备剪贴板。
- `features.multiDeviceClipboard.deviceName`：设备展示名，空值时使用系统主机名。
- `features.multiDeviceClipboard.maxSyncBytes`：最大同步大小，最大 1GB。
- `features.multiDeviceClipboard.historyLimit`：历史记录数量。
- `shortcuts.system.toggleMultiDeviceClipboard`：系统级唤出快捷键，默认 `Alt+V`。
