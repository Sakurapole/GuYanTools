> **版本**：1.0
> **日期**：2026-03-23
> **文档状态**：草案

---

## 1. 项目背景

GuYanTools 桌面端需要集成一个**内置 Todo**功能模块，提供轻量但功能完备的任务管理能力。功能定位参考 **Microsoft To Do**，作为应用的**内置功能**（非插件），使用独立的功能窗口/页面承载。

### 1.1 核心定位

- **轻量化**：不做成重量级项目管理工具，聚焦个人日常 Todo 管理
- **内置集成**：作为 `built-in_plugins` 内置功能模块，与现有插件架构融合
- **数据本地化**：所有数据持久化到本地 SQLite 数据库，确保隐私和离线可用

### 1.2 技术约束

- Electron 37 + Vue 3 + TypeScript + Pinia
- Rust Core (`@guyantools/core`) 提供 SQLite 数据库层
- 遵循现有 IPC 通信模式（主进程 ↔ 渲染进程）
- 遵循现有配置管理模式（`features.todo` 节点存入 `app_config.json`）

---

## 2. 核心概念定义

| 术语             | 定义                                                         |
| ---------------- | ------------------------------------------------------------ |
| **我的一天**     | 每天的焦点任务视图，用户手动添加当天要完成的任务             |
| **列表 (List)**  | 任务的分组/分类容器，用户可自定义创建                       |
| **任务 (Todo)**  | 最小的待办事项单元，包含标题、状态等核心属性                 |
| **步骤 (Step)**  | 任务的子步骤，用于拆分复杂任务为可执行的小步骤               |
| **重复任务**     | 按照指定周期自动重新创建的任务                               |
| **智能列表**     | 系统预置的动态列表，根据条件自动聚合任务（如"重要"、"已计划"） |

## 3. 功能需求

### 3.1 我的一天（My Day）

#### FR-TODO-01：我的一天视图

- 打开 Todo 功能时**默认跳转到"我的一天"**视图
- 展示用户手动添加到"我的一天"的任务列表
- 支持快速添加新任务到当天
- 每天自动重置"我的一天"列表（已完成的移除，未完成的保留并提示）
- 顶部展示日期和个性化问候词

#### FR-TODO-02：跨天未完成提醒

- 当用户打开 Todo 功能时，如果存在**前一天未完成的"我的一天"任务**：
  - 弹出提示卡片/弹窗，列出未完成任务
  - 用户可选择：**继续添加到今天** / **移除出我的一天** / **标记为已完成**
- 提示仅在当天首次打开时触发一次

#### FR-TODO-03：建议任务

- 在"我的一天"底部或侧边展示**建议添加到今天的任务**：
  - 今天到期的任务
  - 标记为重要的未完成任务
  - 以往在每天中出现但未完成的重复任务
- 用户可一键将建议任务添加到"我的一天"

---

### 3.2 列表管理

#### FR-TODO-04：自定义列表

- 支持**创建/重命名/删除**自定义列表
- 每个列表具有**名称**和可选**图标**（复用项目的 SVG 图标库）
- 支持自定义列表的**排序顺序**拖拽调整
- 支持列表的**主题颜色**设置

#### FR-TODO-05：智能列表（系统预置）

- **我的一天**：当天焦点任务
- **重要**：标记为重要的所有任务
- **已计划**：设置了截止日期的所有任务
- **全部**：所有列表中的全部任务
- **已完成**：所有已完成的任务
- 智能列表不可删除，但可设置是否在侧边栏显示

#### FR-TODO-06：列表分组

- 支持创建**分组**，将多个列表归类到同一分组下
- 分组支持折叠/展开
- 分组内列表支持拖拽排序

---

### 3.3 任务管理

#### FR-TODO-07：任务基本操作

- 支持在列表中**快速添加任务**（底部输入框，回车即添加）
- 支持**标记完成 / 取消完成**（复选框）
- 完成的任务以删除线样式展示，并可选择自动移至列表底部
- 支持**删除任务**（右键菜单或详情面板中）
- 支持任务在**不同列表间移动/复制**

#### FR-TODO-08：任务详情（侧边面板）

点击任务标题后，在右侧展开**任务详情面板**，包含以下可配置项：

##### 3.3.1 步骤 (Steps)

- 支持为任务添加多个**子步骤**
- 每个步骤支持独立的完成状态
- 步骤支持拖拽排序
- 展示步骤完成进度（如 2/5）

##### 3.3.2 添加到"我的一天"

- 在详情面板中可快速将任务添加到 / 移出"我的一天"

##### 3.3.3 提醒 (Reminder)

- 支持设置**提醒时间**（绝对时间）
- 预置快捷选项：今天晚些时候、明天、下周
- 支持自定义日期时间选择
- 到达提醒时间时，通过应用的**系统通知窗口**发送提醒

##### 3.3.4 截止日期 (Due Date)

- 支持设置**截止日期**
- 预置快捷选项：今天、明天、下周
- 支持自定义日期选择
- 过期任务以**红色**高亮标注
- 今日到期任务以**橙色**标注

##### 3.3.5 重复 (Repeat)

- 支持设置任务**重复周期**：
  - 每天
  - 每个工作日（周一至周五）
  - 每周
  - 每月
  - 每年
  - 自定义（每 N 天/周/月）
- 当重复任务被标记为完成后，自动生成下一周期的新任务

##### 3.3.6 备注 (Note)

- 支持添加**富文本/Markdown 备注**
- 备注内容在详情面板底部展示
- 支持自动保存

##### 3.3.7 重要标记 (Important)

- 支持将任务标记为**重要**（星标）
- 标记重要的任务自动出现在"重要"智能列表中

---

### 3.4 排序与筛选

#### FR-TODO-09：任务排序

- 支持以下排序方式：
  - 重要性优先
  - 截止日期
  - 创建日期
  - 字母顺序
  - 自定义拖拽排序
- 已完成任务始终排在未完成任务之后

#### FR-TODO-10：任务搜索

- 支持全局搜索：按任务标题、备注内容匹配
- 搜索结果高亮展示匹配内容
- 支持在当前列表内搜索

---

### 3.5 通知与提醒

#### FR-TODO-11：提醒通知

- 到达提醒时间时，通过应用的系统通知窗口（已有 `notification_ipc`）推送
- 通知内容包含：任务标题、所属列表、截止日期
- 点击通知可跳转到对应任务详情
- 应用处于最小化/后台运行时仍可触发通知

---

### 3.6 设置与配置

#### FR-TODO-12：Todo 功能配置

- 在设置页面（`Settings.vue`）中增加 Todo 功能配置面板
- 配置项包括：
  - 智能列表的显示/隐藏
  - 默认新任务添加到的列表
  - 完成音效开关
  - 确认删除开关
  - "我的一天"每日重置逻辑配置

## 4. 非功能需求

### 4.1 性能

| 指标                   | 目标           |
| ---------------------- | -------------- |
| 列表切换响应时间       | < 100ms        |
| 任务列表渲染（1000条） | < 300ms        |
| 任务创建/修改持久化     | < 50ms         |
| 搜索响应时间           | < 200ms        |
| 重复任务生成           | 后台异步完成   |

### 4.2 可用性

- 支持键盘快捷键操作（Tab 切换、Enter 添加/确认、Delete 删除）
- 支持拖拽排序（列表、任务、步骤）
- 响应式布局，适配不同窗口尺寸

### 4.3 数据安全

- 所有数据存储在本地 SQLite，不涉及网络传输
- 支持数据导出（JSON 格式）备份
- 删除操作需二次确认（可配置）

### 4.4 国际化

- 所有 UI 文本通过 `i18n` 系统管理，支持中英文切换

## 5. 用户故事

### US-01：我的一天工作流

**作为**用户，**我想**每天打开 Todo 时看到今天的重点任务并收到昨日未完成的提醒，**以便**高效规划每天的工作。

### US-02：快速添加任务

**作为**用户，**我想**在列表底部快速输入任务标题并回车添加，**以便**最小化操作步骤，不打断思路。

### US-03：管理复杂任务

**作为**用户，**我想**为一个任务添加步骤、截止日期和提醒，**以便**将大任务拆分为可执行的小步骤并按时完成。

### US-04：周期性任务

**作为**用户，**我想**设置每周重复的任务，**以便**不用手动重复创建常规性任务。

### US-05：组织任务结构

**作为**用户，**我想**创建多个列表和分组来分类管理我的任务，**以便**保持待办事项的有序和清晰。
