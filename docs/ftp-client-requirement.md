## 1. 概述

### 1.1 目标

在 GuYanTools 桌面端应用中实现一个完整的 SFTP / FTP 文件传输客户端，复用已有的 SSH 连接基础设施（`SshConnectionManager`、`SshProfile` 等），为用户提供类似 WinSCP / FileZilla 的文件管理与传输体验。

### 1.2 适用范围

- **协议支持**：SFTP（基于 SSH）、FTP、FTPS（FTP over TLS）
- **平台**：Windows（当前主要目标），架构预留跨平台扩展能力
- **集成方式**：作为应用内置功能（built-in plugin），与 SSH 终端、设置页等模块深度集成

### 1.3 核心价值

| 价值维度 | 说明 |
|---------|------|
| 统一入口 | SSH 终端 + 文件传输在同一应用中无缝切换 |
| 连接复用 | SFTP 连接可直接使用已配置的 SSH Profile，无需重复配置 |
| 高效传输 | 多任务并行、断点续传、定时任务等企业级传输能力 |
| 安全可靠 | 复用 SSH 密钥管理体系，支持 CA 认证、Jump Host 代理 |

## 3. 连接管理

### 3.1 连接创建

#### FR-CONN-001：新建 SFTP/FTP 连接

- 用户可创建新的 SFTP 或 FTP/FTPS 连接
- 需录入以下信息：
  - **主机地址**（Host）
  - **端口**（Port，SFTP 默认 22，FTP 默认 21）
  - **协议类型**（SFTP / FTP / FTPS）
  - **用户名**（Username）
  - **认证方式**（密码 / 公钥 / Keyboard-Interactive）
- 表单应根据协议类型动态调整字段（如 FTP 无需公钥选项）

#### FR-CONN-002：复用 SSH 配置

- SFTP 连接可选择"使用已有 SSH 配置"
- 从已配置的 `SshProfile` 列表中选择，自动填充主机、端口、用户名、认证方式等字段
- 复用 SSH 配置时，应自动继承 Jump Host 代理设置
- FTP/FTPS 连接也可以选择使用 SSH 隧道方式，此时需指定一个 SSH 配置作为隧道

### 3.2 会话管理

#### FR-CONN-003：保存连接为会话

- 用户可将已配置的连接参数保存为命名会话（Session）
- 保存后可在会话列表中快速选择并重新打开
- 会话信息持久化至 SQLite 数据库

#### FR-CONN-004：文件夹/树形结构管理会话

- 会话支持以文件夹方式组织，形成树形结构
- 支持创建、重命名、删除文件夹
- 支持拖拽调整会话在文件夹间的位置
- 树形结构示例：
  ```
  ├── 生产服务器
  │   ├── Web Server 01
  │   ├── Web Server 02
  │   └── Database Server
  ├── 测试环境
  │   ├── Staging
  │   └── QA
  └── 个人服务器
      └── VPS
  ```

#### FR-CONN-005：应用重启恢复连接

- 系统在正常退出前，保存当前所有已打开的连接状态（Profile ID、远程路径、本地路径、标签页顺序等）
- 应用重新启动时，自动恢复上次退出前已打开的连接
- 恢复时应尝试重新建立连接，若连接失败则显示"断开"状态并提供重连按钮

## 4. 文件传输引擎

### 4.1 基础传输

#### FR-TRANS-001：文件上传与下载

- 支持本地到远程的文件上传
- 支持远程到本地的文件下载
- 支持单文件和批量文件/文件夹传输
- 文件夹传输时递归处理子目录结构

#### FR-TRANS-002：拖拽传输

- 支持从本地文件浏览器拖拽文件到远程文件浏览器发起上传
- 支持从远程文件浏览器拖拽文件到本地文件浏览器发起下载
- 支持从 Windows Explorer 拖拽文件到应用窗口发起上传
- 支持从应用窗口拖拽文件到 Windows Explorer 发起下载
- 拖拽时应有视觉反馈指示目标区域

### 4.2 并行与队列

#### FR-TRANS-003：多任务并行传输

- 支持多个传输任务同时执行，提升带宽利用率
- 可配置最大并行任务数（默认 3，建议范围 1~10）
- 每个连接可独立配置并行通道数

#### FR-TRANS-004：任务队列管理

- 提供统一的任务队列视图
- 显示任务状态：等待中 / 传输中 / 已暂停 / 已完成 / 失败
- 支持队列内任务排序（按添加时间、文件大小、文件名）
- 支持设置排序规则
- 显示每个任务的：
  - 文件名 / 路径
  - 文件大小
  - 传输进度（百分比 + 已传输大小）
  - 传输速度
  - 预计剩余时间
  - 来源 → 目标路径

#### FR-TRANS-005：任务优先级

- 支持设置任务优先级，至少包含：高 / 中 / 低
- 下载任务必须支持优先级设定
- 高优先级任务在队列中优先调度执行
- 支持运行中手动调整任务优先级

### 4.3 暂停与恢复

#### FR-TRANS-006：手动暂停和恢复

- 支持暂停单个传输任务
- 支持暂停全部传输任务
- 暂停后任务状态保持，可随时恢复继续传输
- 恢复传输时从断点位置继续，而非重新开始

#### FR-TRANS-007：断点续传

- 文件下载支持断点续传
- 网络中断或传输失败后，重新连接时可从已传输的位置继续
- 通过文件大小校验确保续传数据完整性
- 对于 SFTP 协议，利用 `seek` 定位到断点位置
- 对于 FTP 协议，利用 `REST` 命令实现续传

#### FR-TRANS-008：失败重试

- 网络中断或传输失败时自动重试
- 可配置最大重试次数（默认 3 次）
- 可配置重试间隔（默认 5 秒，指数退避策略）
- 重试失败后任务标记为"失败"，保留在队列中供用户手动重试

## 5. 文件浏览与管理

### 5.1 双栏布局

#### FR-BROWSE-001：本地与远程目录双栏展示

- 界面采用左右双栏布局
- 左栏显示本地文件系统目录
- 右栏显示远程服务器目录
- 每栏支持：
  - 路径导航栏（面包屑 + 手动输入）
  - 文件/文件夹列表（支持列表和详情两种视图）
  - 显示文件名、大小、修改日期、权限、所有者
  - 排序功能（按文件名、大小、日期等）

#### FR-BROWSE-002：联动导航

- 支持本地/远程目录联动导航模式
- 开启联动后，在一侧进入某目录时，另一侧自动跳转到同名目录（若存在）
- 支持远程/远程目录联动导航（双远程面板模式）
- 联动模式可随时开关

### 5.2 文件操作

#### FR-BROWSE-003：基础文件操作

- 支持以下远程文件/目录操作：
  - **新建目录**：创建远程文件夹
  - **删除**：删除远程文件或文件夹（需确认提示）
  - **重命名**：重命名远程文件或文件夹
  - **修改权限**：修改远程文件/文件夹权限（chmod），提供数字模式和可视化复选框两种方式
- 所有操作通过右键菜单和工具栏均可触发

#### FR-BROWSE-004：图片缩略图预览

- 远程目录中的图片文件支持缩略图预览
- 支持常见图片格式：jpg、png、gif、bmp、webp、svg
- 缩略图异步加载，不阻塞文件列表渲染
- 可在设置中关闭缩略图功能以节省带宽

#### FR-BROWSE-005：多本地工作目录

- 本地文件浏览器支持维护多个工作目录
- 用户可添加、移除常用本地目录作为"书签"
- 提供快速切换下拉菜单

### 5.3 搜索与过滤

#### FR-BROWSE-006：快速搜索

- 支持在当前目录中快速搜索文件和文件夹
- 实时过滤匹配（模糊搜索 / 通配符匹配）
- 搜索结果高亮显示

#### FR-BROWSE-007：文件过滤

- 支持基于规则过滤文件展示（如按扩展名、大小、日期）
- 过滤规则支持组合条件（AND / OR）
- 支持保存常用过滤条件为预设
- 支持快速复用已保存的过滤预设
- 内置常用预设：
  - 仅显示文件 / 仅显示文件夹
  - 隐藏隐藏文件
  - 按扩展名（如 `.log`、`.conf`）

## 6. 目录比较与同步

#### FR-SYNC-001：目录差异比较

- 支持比较两个目录间的文件差异
- 比较维度：
  - 文件存在性（仅在左侧 / 仅在右侧 / 双方都有）
  - 文件大小差异
  - 修改时间差异
  - 文件内容差异（可选，基于校验和）
- 差异结果以颜色编码展示：
  - 🟢 绿色：新增文件
  - 🔴 红色：待删除文件
  - 🟡 黄色：修改过的文件
  - ⚪ 白色/灰色：相同文件

#### FR-SYNC-002：同步前预览

- 在执行同步前，展示详细的差异报告
- 列出将要执行的每一个操作（上传 / 下载 / 删除 / 跳过）
- 用户可逐项确认或取消
- 显示统计摘要（新增 N 个文件、修改 N 个文件、删除 N 个文件、总传输大小）

#### FR-SYNC-003：一键同步

- 确认后支持一键执行同步操作
- 同步方向支持：
  - 本地 → 远程
  - 远程 → 本地
  - 双向同步（以新版本为准）
- 同步过程中显示进度，支持取消

## 7. 远程文件编辑

#### FR-EDIT-001：直接编辑远程文件

- 支持直接打开远程文件进行编辑
- 内置简单文本编辑器用于轻量级编辑
- 编辑保存后自动将文件上传回服务器

#### FR-EDIT-002：调用外部编辑器

- 支持配置并调用用户指定的外部编辑器（如 VS Code、Notepad++、Vim 等）
- 工作流程：
  1. 将远程文件下载到本地临时目录
  2. 使用指定编辑器打开临时文件
  3. 监控文件变更（File Watcher）
  4. 编辑完成后自动将修改后的文件上传回服务器
- 可配置默认编辑器路径
- 可配置是否在编辑器关闭后自动清理临时文件

## 8. 安全与认证

### 8.1 认证方式

#### FR-AUTH-001：多认证方式支持

- **密码认证**：明文密码输入，支持可选记住密码（加密存储）
- **公钥认证**：指定私钥文件路径，支持带密码短语的私钥
- **Keyboard-Interactive 认证**：支持服务器端交互式认证流程（如二步验证）

#### FR-AUTH-002：SSH 密钥管理

- 支持 SSH 密钥生成（RSA / Ed25519 / ECDSA）
- 支持密钥导入（从文件 / 从剪贴板）
- 支持密钥导出
- 提供密钥管理界面，列出所有已管理的密钥
- 密钥存储加密保护

### 8.2 主机验证

#### FR-AUTH-003：主机密钥校验

- 首次连接新主机时，提示用户确认服务器指纹
- 复用现有 `SshKnownHost` 机制进行主机密钥校验
- 主机密钥变更时发出安全警告
- 提供主机密钥管理界面，可查看、删除已信任的指纹

### 8.3 代理与 CA

#### FR-AUTH-004：Jump Host 代理

- 支持通过 Jump Host（跳板机）方式访问目标主机
- 复用现有 SSH Profile 中的 `SshJumpHost` 配置
- 文件传输通道也通过 Jump Host 建立

#### FR-AUTH-005：OpenSSH CA 认证

- 支持使用 OpenSSH Certificate Authority 签发的证书进行认证
- 支持主机 CA 证书和用户 CA 证书

## 9. 界面与交互

### 9.1 多标签页管理

#### FR-UI-001：多标签页

- 支持在多个标签页中同时管理多个文件传输连接
- 每个标签页独立维护一组本地/远程文件浏览器
- 标签页支持拖拽重排
- 标签页显示连接名称和状态指示器（已连接 / 断开）

#### FR-UI-002：标签组并行显示

- 支持将标签页拆分为多组，并行排列显示
- 支持水平和垂直方向拆分
- 方便同时操作多个连接

### 9.2 面板布局

#### FR-UI-003：可停靠面板

- 界面采用可停靠、可隐藏的面板布局
- 主要面板：
  - **会话管理面板**：树形会话列表（左侧/可隐藏）
  - **本地文件面板**：本地文件浏览器
  - **远程文件面板**：远程文件浏览器
  - **传输队列面板**：传输任务列表（底部/可隐藏）
  - **日志面板**：操作日志输出（底部/可隐藏）
- 面板可调整大小、可隐藏/显示

### 9.3 与 SSH 终端集成

#### FR-UI-004：SSH 终端联动

- 在文件传输连接的右键菜单或工具栏中，提供"在终端中打开"选项
- 点击后自动创建一个与当前连接相同主机的 SSH 终端标签
- 复用相同的 SSH Profile 和认证信息
- 终端打开时自动 `cd` 到当前浏览的远程目录
- 反向支持：从 SSH 终端界面快速切换到同主机的文件管理器

- 支持从 Windows 资源管理器复制文件并粘贴到远程面板

## 11. 计划任务与自动化

#### FR-AUTO-001：定时传输任务

- 支持按用户定义时间自动执行上传或下载任务
- 定时策略支持：
  - 单次执行（指定日期时间）
  - 周期性执行（每小时 / 每天 / 每周 / 自定义 cron 表达式）
- 定时任务配置项：
  - 源路径（本地或远程）
  - 目标路径（本地或远程）
  - 传输方向
  - 冲突处理策略（覆盖 / 跳过 / 重命名）
  - 是否包含子目录
- 定时任务列表可查看下次执行时间、上次执行结果

#### FR-AUTO-002：命令行触发

- 支持通过命令行方式触发传输任务
- 命令行参数设计（示例）：
  ```bash
  guyantools ftp upload --session "Web Server 01" --local "/path/to/file" --remote "/var/www/"
  guyantools ftp download --session "Web Server 01" --remote "/var/log/app.log" --local "./logs/"
  guyantools ftp sync --session "Web Server 01" --local "./dist/" --remote "/var/www/html/" --direction upload
  ```
- 支持通过 IPC 与已运行的主应用实例通信
- 命令行执行结果输出到标准输出

## 12. 高级功能

#### FR-ADV-001：FXP（服务器对传）

- 支持 FXP（File eXchange Protocol）——两个 FTP 服务器之间直接传输文件
- 要求两端都为 FTP 连接
- 传输数据不经过客户端，直接在两台服务器间流动
- UI 中提供双远程面板操作模式支持 FXP

## 13. 数据持久化

### 13.1 数据库表结构设计

所有持久化数据存储于应用的 SQLite 数据库中，与现有 SSH 模块共用同一数据库实例。

#### 连接会话表 `ftp_sessions`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | UUID |
| label | TEXT | 会话显示名称 |
| protocol | TEXT | 'sftp' / 'ftp' / 'ftps' |
| host | TEXT | 主机地址 |
| port | INTEGER | 端口号 |
| username | TEXT | 用户名 |

## 14. 非功能性需求

### 14.1 性能

| 指标 | 要求 |
|------|------|
| 文件列表加载 | 1000 个文件内应在 2 秒内完成渲染 |
| 传输吞吐量 | 单连接传输速度不低于网络带宽的 80% |
| 并行任务数 | 至少支持 10 个并行传输任务 |
| 内存占用 | 空闲状态下模块额外内存占用不超过 50MB |
| 大文件支持 | 支持单文件最大 50GB 以上传输 |

### 14.2 可靠性

- 传输任务状态持久化，应用崩溃后可恢复未完成任务
- 连接断开时自动重连（若已配置）
- 文件传输完整性校验（可选 MD5/SHA256）

### 14.3 安全性

- 凭据加密存储，与现有 SSH 模块共用加密机制（`encrypt_credential` / `decrypt_credential`）
- 传输通道加密（SFTP 基于 SSH、FTPS 基于 TLS）
- 不在日志中输出明文密码或密钥

### 14.4 可扩展性

- Rust 后端采用 trait-based 协议适配器设计，便于未来扩展 SCP、WebDAV 等协议
- UI 组件设计模块化，支持通过配置控制功能可见性
- 传输引擎与协议层解耦

### 14.5 国际化

- 所有界面文本通过 i18n 系统管理
- 初期支持中文和英文

### 16.3 实现备注

- 本轮后端新增了 `ftp` 模块、NAPI 导出与数据库迁移，当前已支持 `SFTP`、`FTP`、`FTPS` 的直连会话建链，以及 `FTP/FTPS` 通过 `SshProfile` 建立 SSH 隧道。
- 会话管理侧已补齐基础文件夹树，当前支持通过配置归档到文件夹并在侧栏树形展示，也支持在侧栏树中拖拽调整会话归属和基础顺序、拖拽文件夹切换父级。
- UI 侧已补充活动连接标签页、拖拽重排、水平/纵向工作区切换、主/副两个标签组并行显示，以及侧栏/本地/远程/任务/日志面板的显示隐藏、停靠位置切换与尺寸持久化。
- 文件浏览当前已补充路径面包屑导航、列表 / 详情视图切换和文件名 / 大小 / 修改时间 / 权限 / 所有者展示；联动导航也已支持在“本地 ↔ 远程”和“远程 ↔ 远程”两种模式间切换。对远程 `chmod` 同时支持数字模式输入与复选框模式，规则过滤也已扩展到大小、最近修改时间、AND / OR 组合，以及内置过滤预设库。
- 缩略图当前已补充单张最大读取体积、每侧预加载数量和基础内存缓存裁剪，便于在大目录和低带宽场景下控制预览开销。
- 已打开会话当前会持续同步到 `ftp_restore_state`，应用再次进入 FileTransfer 页面时会尝试自动恢复；恢复失败的项会在侧栏单独列出。
- 当 FTP 配置复用的 SSH Profile 含有 `SshJumpHost` 时，当前已可通过单层 Jump Host 打通 SFTP 建链和 FTP/FTPS 隧道链路；既支持引用现有 SSH Profile 的 `profileId` 模式，也支持使用 `host` / `port` / `username` / `authType` 的手工配置模式，手工模式会复用当前 SSH Profile 的密码或私钥凭据完成跳板认证。
- 主机密钥校验当前已补充完整的 FTP 页管理入口：首次连接会提示确认服务器指纹，已信任主机指纹可在传输设置中查看、刷新和删除；若服务端主机密钥与已存指纹不一致，会显示明确的安全警告并要求用户显式确认是否覆盖信任。
- 已补充独立的 SSH 密钥管理库：主应用侧会把托管私钥加密存入 `ssh_managed_keys`，并提供 Ed25519 / ECDSA / RSA 生成、从文件或剪贴板导入、导出私钥及同名 `.pub` 公钥、复制公钥和删除的集中管理界面，供 SSH / SFTP 配置录入前统一维护密钥资产。
- OpenSSH CA 认证当前已完整覆盖用户证书与主机证书两条链路：SSH Profile 与直接 SFTP Profile 都可保存 `private key` 对应的 `OpenSSH user certificate` 路径，并在 SSH 终端、SFTP 直连、SFTP 复用 SSH Profile、FTP/FTPS 的 SSH 隧道、Jump Host 复用私钥凭据等链路里使用 `authenticate_openssh_cert` 完成用户证书认证；同时已为 SSH / SFTP 目标主机和手工 Jump Host 增加 `Host CA public key path` 配置，握手期会基于本地 vendor 的 `russh` 补丁保留原始 host certificate blob，并结合 `ssh-key` 对证书类型、有效期、principal、critical options 和 CA 签名链执行校验。
- 连接期认证当前已支持 `Keyboard-Interactive` 多轮提示：后端会保留待认证会话并把服务端 prompts 回传到前端对话框，用户逐项填写后可继续完成同一次连接。
- `FTP/FTPS` 当前通过 `suppaftp` 建立控制会话，已支持直连和经 `SshProfile` 的 SSH 隧道两种建链方式；远程目录浏览、基础目录操作、文本读取/保存、图片预览，以及纳入统一任务队列的上传/下载都已覆盖。
- `FTP/FTPS` 的 SSH 隧道当前会复用所选 `SshProfile` 的认证信息与 Jump Host 配置，并把控制连接及被动模式数据连接统一转发到目标 FTP 主机；若隧道 SSH Profile 使用密码认证，当前要求密码已保存在 SSH 模块中。
- 传输任务以异步任务方式执行，当前已支持 `SFTP` / `FTP` / `FTPS` 的单目录递归、多选批量触发、统一队列面板、排序与优先级调度、手动暂停/恢复、失败后的手动断点续传（`SFTP` 用 `seek`、`FTP/FTPS` 用 `REST`），以及可配置最大次数和基础等待秒数的自动重试。
- `FXP` 与**远程中继拷贝**当前已接入统一传输队列，并在双远程面板模式下提供操作入口。两端均为直连纯 `FTP` 时走真正的服务端 `PASV/EPSV + PORT/EPRT + STOR + RETR` 服务器对传（任务方向 `fxp`）；其他协议组合（`SFTP↔SFTP`、`SFTP↔FTP/FTPS`、经 SSH 隧道的 `FTP` 等）则自动降级为客户端中继模式（任务方向 `remote_copy`），以 256 KB 缓冲区流式管道将源端数据读取并写入目标端，支持逐文件暂停检查点与目录递归；前端按协议组合自动识别，按钮标签在 FXP 与"远程复制"之间切换，并统一通过 `fxp_transfer` NAPI 入口调度。
- 远程拖出到 Explorer 当前会在主进程下把选中的远程文件/目录导出到本地临时 staging 目录，再通过 Electron 原生 `startDrag` 发起文件拖拽；同一链路也可被应用内本地面板接住并复制到当前本地目录。
- 通知系统当前已支持在传输完成 / 失败后通过自定义通知窗口提醒，并在点击通知后拉起主窗口、跳转到 FileTransfer 页面并展开对应任务。
- 已新增 Windows 右键菜单注册表安装/卸载能力，并支持从 Explorer 右键“通过 GuYanTools 上传”把路径导入主应用，在 FileTransfer 页面中确认上传到当前远程目录；当前静态菜单注册项已启用多选模型，可一次接收多个 Explorer 选中文件或目录。
- Explorer 协同当前已支持读取剪贴板中的 Windows 文件路径，并在当前激活的远程面板通过工具栏“粘贴上传”、`Ctrl/Cmd+V` 或 `Shift+Insert` 触发上传；同时支持右键菜单导入后的待上传确认、复制本地/远程条目信息到剪贴板，以及远程条目拖出到 Explorer。
- 已补充 FTP 自动化与 CLI 触发能力：当前支持单次 / 每小时 / 每天 / 每周 / 自定义五段式 Cron 的计划传输任务、最近执行结果展示、立即执行，以及基础冲突策略（冲突时跳过 / 允许并行）；计划任务状态会版本化持久化到本地状态文件，启动时会自动恢复并把异常中断前处于 running 的任务修正为失败待查。`ftp upload` / `download` / `sync` 命令均可通过单实例机制转发给已运行主应用执行，其中 `sync` 已支持 `localToRemote` / `remoteToLocal` / `bidirectional` 三种方向、递归比较、可选内容校验与四种冲突策略，并会输出按操作类型聚合的标准输出摘要；次级 CLI 进程会等待主实例结果并透传标准输出 / 标准错误和退出码（`0` 成功、`2` 用法错误、`3` 运行时失败）。
- 同步预览当前已支持逐项勾选确认或取消后再执行，并可切换到递归比较模式展示嵌套相对路径；当前已能基于递归比较结果执行递归同步，并支持基础冲突策略切换。
- 外部编辑当前已支持自定义编辑器路径，并在使用自定义编辑器时支持编辑器关闭后清理临时文件；系统默认编辑器仍无法严格跟踪关闭时机。
- 布局侧已新增日志面板，可展示最近的连接与任务事件，并支持独立显示/隐藏；会话侧栏现可左右停靠并调整宽度，传输队列与日志面板可切换到底部或右侧辅助停靠区，尺寸偏好会本地持久化。
- 任务面板当前已增加基于实时速度的 ETA 展示；标签页当前已补充“关闭其他 / 关闭右侧 / 恢复最近关闭”的基础生命周期操作。
- SSH 联动当前已补充双向切换：文件传输页可打开联动终端并自动 `cd` 到当前远程目录，SSH 终端工具栏也可反向切到复用同一 SSH Profile 的文件管理器视图。
- 原生构建已通过 `cargo build --features napi` 验证；`pnpm build:debug` 在覆盖现有 `.node` 产物时因文件占用失败，不属于代码编译错误。
