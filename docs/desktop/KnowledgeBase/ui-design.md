# 知识库 UI 设计

> 版本：0.1
> 日期：2026-05-27
> 文档状态：草案

## 1. 页面结构

知识库采用工具型三栏布局，不做营销式首页。

```text
顶部工具栏
├─ 当前库/标题
├─ 搜索框
└─ 新建页面 / 导入 / 速记

左侧资料树
├─ 快速收集箱
├─ 最近编辑
├─ 收藏
└─ 知识库/空间/文件夹/页面

中间工作区
└─ 页面编辑器或文档预览

右侧 Inspector
├─ 目录
├─ 反链
├─ 标签
├─ 附件
├─ Todo
└─ AI
```

## 2. V0.1 页面

V0.1 只提供空状态骨架：

- 顶部工具栏。
- 左侧资料树占位。
- 中间空状态。
- 右侧 Inspector 占位。

按钮处于 disabled 状态，避免用户误以为功能已经可用。

## 3. V0.3 页面

V0.3 接入真实数据：

- 左侧树展示默认库、默认空间和快速收集箱。
- 支持新建空间、文件夹、Markdown 页面。
- 支持选中页面后加载中间区。
- 右侧 Inspector 展示页面基础属性。

## 4. 交互原则

- 资料树密度要高，适合长期扫描。
- 页面编辑区优先给内容留空间。
- 右侧 Inspector 可折叠。
- 搜索入口始终在顶部可见。
- 速记入口始终可一键触达。

## 5. 空状态文案

空状态只说明当前状态，不解释完整功能：

- 主标题：等待创建第一条知识内容。
- 副文本：库、空间、页面、附件、标签和反链结构已预留。

后续真实功能上线后，空状态改为直接动作：

- 新建 Markdown 页面。
- 导入文件。
- 打开速记。

## 6. 响应式

- 宽屏：三栏。
- 中等宽度：隐藏右侧 Inspector。
- 窄屏：隐藏左侧资料树，只保留中间区。

桌面端以常规窗口尺寸为主，移动适配不是首要目标。

## 7. 组件和主题约束

- 操作按钮、输入框、字段、弹窗、菜单、Tab、图标按钮优先使用主应用已有通用 UI 组件，不在知识库页面内重复造基础控件。
- 浏览器原生 `prompt()`、`confirm()`、`alert()` 不适用于 Electron 当前渲染环境；需要输入文本时使用全局 `TextPromptDialog`，需要危险确认时使用全局确认弹窗。
- 如果发现缺少可复用基础控件，且该控件不属于知识库专有业务，应放入 `desktop/src/windows/main/components/ui/` 和 `desktop/src/windows/main/composables/`，再由知识库调用。
- 知识库样式必须跟随应用主题变量，不硬编码单一深色或浅色方案。优先使用 `--ui-text-*`、`--ui-surface-*`、`--ui-border-*`、`--ui-button-*`、`--ui-input-*`、`--primary-color` 和 `--background-color`。
- 页面根节点可以为编辑器子组件提供局部别名，但别名必须映射回应用主题变量，不能形成独立主题系统。
