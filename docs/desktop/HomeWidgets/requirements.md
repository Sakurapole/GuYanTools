# 首页内置小组件需求文档

> 版本：1.0
> 日期：2026-03-25
> 文档状态：草案

---

## 1. 背景

当前首页小组件区域已经具备以下基础能力：

- 固定网格布局与自动重排
- 小组件拖拽、删除、编辑、持久化
- 创建时选择宽高占位
- 通用卡片式展示和跳转动作配置

但现有实现仍然是“通用快捷卡片”模型，不支持“内置业务型小组件”：

- 创建入口只能先选尺寸，不能先选小组件类型
- 数据模型没有 `widgetType`、`sizePreset`、`widgetConfig`
- 渲染层只有统一的图标 + 标题卡片视图
- 1x1 尺寸仍然可选，不符合本次需求

本次目标是把首页小组件升级为“类型化内置组件系统”，首期覆盖：

- 番茄钟
- 日期
- 天气

---

## 2. 目标

### 2.1 产品目标

- 用户在首页创建小组件时，直接选择“小组件类型 + 可用尺寸 + 配置项”
- 不同内置小组件仅提供适合自身能力的尺寸，不开放 1x1
- 同一个小组件在不同尺寸下呈现不同的信息密度和布局
- 保留当前通用快捷卡片能力，避免影响已有首页使用习惯

### 2.2 技术目标

- 在现有首页布局系统上演进，不推翻现有 `CompArea` 网格逻辑
- 扩展持久化模型，支持小组件类型、尺寸预设、业务配置
- 建立内置小组件注册表，后续可继续新增更多内置组件

---

## 3. 范围

### 3.1 本期范围

- 首页新增“内置小组件创建流程”
- 新增番茄钟、日期、天气 3 类内置小组件
- 每类小组件支持多尺寸适配
- 新增小组件配置表单、尺寸预览、效果预览
- 支持编辑时修改配置和切换到该类型允许的尺寸
- 天气小组件支持基础城市配置、天气展示和异常兜底

### 3.2 不在本期范围

- 1x1 小组件
- 插件小组件接入首页类型化系统
- 用户自定义任意尺寸模板
- 天气多城市轮播、地图、空气质量等复杂能力
- 日期小组件中的完整日程管理能力
- 番茄钟跨设备同步

基于当前代码，以下模块会直接受到影响：

- `desktop/src/renderer/pages/Home/Home.vue`
- `desktop/src/renderer/components/CompArea/CompArea.vue`
- `desktop/src/renderer/components/CompArea/GridItem.vue`
- `desktop/src/renderer/components/CompArea/WidgetSizePicker.vue`
- `desktop/src/renderer/types/grid.d.ts`
- `desktop/src/shared/home_layout.ts`
- `desktop/src/main/home_layout_ipc.ts`
- `multi_platform_core/src/models/home_layout.rs`
- `multi_platform_core/migrations/004_add_home_layout.sql`

现状判断：

- `CompArea.vue` 右键空白区域后直接打开尺寸选择器，创建的是通用 `GridItem`
- `GridItem.vue` 只有卡片式通用渲染，没有业务组件插槽
- `home_layout` 相关 DTO 和 Rust model 没有组件类型和配置字段
- 当前 `WidgetSizePicker.vue` 默认支持 1x1 到 6x6 的自由组合，不符合本次设计

## 5. 核心概念

| 术语 | 定义 |
| --- | --- |
| 小组件来源 | `builtin`、`shortcut`、`plugin`，本期重点是 `builtin` |
| 小组件类型 | 业务类型，如 `pomodoro`、`date`、`weather` |
| 尺寸预设 | 某个类型允许的固定尺寸方案，如 `2x2`、`4x2`、`4x3` |
| 小组件配置 | 与业务相关的静态配置，使用 JSON 持久化 |
| 运行时状态 | 组件临时状态，如番茄钟剩余时间；不要求全部进入首页布局表 |

## 6. 用户流程

### 6.1 创建流程

1. 用户在首页小组件区域空白处右键，点击“新建组件”
2. 打开新的“小组件创建弹窗”，优先展示“内置组件”
3. 用户选择组件类型，例如“番茄钟”
4. 系统展示该类型支持的尺寸卡片，仅展示允许的尺寸，不显示 1x1
5. 用户选择尺寸后，右侧或下方展示对应尺寸的预览
6. 用户填写该组件的配置项
7. 点击确认后，系统自动寻找可放置位置并创建组件

### 6.2 编辑流程

1. 用户右键已有组件，点击“编辑组件”
2. 若为内置组件，编辑弹窗显示：
   - 基础信息
   - 尺寸选项
   - 业务配置
   - 样式配置（如允许）
3. 若切换尺寸，只能切换到当前组件类型支持的尺寸
4. 若切换后当前位置不可用，沿用现有 reflow 逻辑重新布局

### 6.3 兼容流程

- 已有旧组件默认视为 `shortcut` 类型
- 旧数据没有 `widgetType` 时，渲染为当前通用卡片，不强制迁移成内置组件

## 7. 功能需求

### FR-HW-01：类型化创建入口

- 原“仅选尺寸”弹窗升级为“类型 + 尺寸 + 配置”创建弹窗
- 创建弹窗至少支持两个分组：
  - 内置组件
  - 快捷卡片
- 本期允许“插件组件”入口保留为占位，不要求可用

### FR-HW-02：尺寸预设约束

- 所有内置组件都不提供 1x1
- 每种组件仅展示可用尺寸，不允许自由输入宽高
- 尺寸展示需使用视觉卡片或网格预览，而不是纯下拉框

### FR-HW-03：自适应渲染

- 每种组件必须根据尺寸输出不同布局，而不是简单整体缩放
- 组件需要定义最小信息集和扩展信息集
- 尺寸越大，可展示的信息越完整

### FR-HW-04：配置化创建

- 用户在创建时即可配置业务参数
- 配置项需要有默认值，避免弹窗过重
- 配置项修改后应即时刷新预览

### FR-HW-05：持久化模型扩展

- 首页组件数据需要新增以下能力：
  - 组件来源
  - 组件类型
  - 尺寸预设标识
  - 业务配置 JSON
- 保留当前位置、背景、跳转动作等通用字段

### FR-HW-06：旧组件兼容

- 旧数据读取不报错
- 没有新字段的 widget 默认按 `shortcut` 处理
- 已有编辑、删除、拖拽能力继续可用

### FR-HW-07：天气数据兜底

- 天气组件在网络不可用、城市未配置、接口失败时，必须显示明确状态
- 不允许出现空白卡片或卡死加载
- 至少提供以下状态：
  - 未配置城市
  - 加载中
  - 加载失败
  - 最近一次缓存数据

### FR-HW-08：番茄钟状态准确性

- 番茄钟计时应基于时间戳而不是纯 `setInterval` 递减
- 页面切换、窗口失焦后返回时，剩余时间应自动校正
- 到时后支持通知提醒

## 8. 内置小组件规格

### 8.1 番茄钟

#### 支持尺寸

| 尺寸 | 场景 | 展示内容 |
| --- | --- | --- |
| 2x2 | 轻量专注卡 | 剩余时间、当前模式、开始/暂停 |
| 4x2 | 标准工作卡 | 剩余时间、模式切换、开始/暂停/重置、今日完成番茄数 |
| 4x3 | 完整专注卡 | 剩余时间、模式切换、今日统计、轮次进度、快捷配置入口 |

#### 配置项

- 工作时长，默认 25 分钟
- 短休息时长，默认 5 分钟
- 长休息时长，默认 15 分钟
- 长休触发轮次，默认 4
- 完成后是否自动进入下一阶段
- 到时是否发送通知
- 到时是否播放提示音

#### 交互要求

- 支持开始、暂停、重置
- 支持切换当前模式：专注 / 短休 / 长休
- 大尺寸显示统计信息，小尺寸仅保留核心动作

### 8.2 日期

#### 支持尺寸

| 尺寸 | 场景 | 展示内容 |
| --- | --- | --- |
| 2x2 | 今日信息卡 | 年月日、星期 |
| 4x2 | 周期概览卡 | 年月日、星期、当月进度、小型日期矩阵或周摘要 |
| 4x3 | 月历卡 | 当前月份日历、今日高亮、顶部日期信息 |

#### 配置项

- 展示模式，默认“今天优先”
- 一周起始日，默认周一
- 是否显示农历，二期可扩展
- 是否显示节假日，二期可扩展
- 是否显示周数

#### 交互要求

- 2x2 不显示复杂日历结构
- 4x3 才显示完整月历
- 日期切换在本地完成，不依赖网络

### 8.3 天气

#### 支持尺寸

| 尺寸 | 场景 | 展示内容 |
| --- | --- | --- |
| 2x2 | 当前天气卡 | 城市、天气图标、温度、体感/天气文案 |
| 4x2 | 今日天气卡 | 当前天气、最高/最低温、未来数小时简报 |
| 4x3 | 短期预报卡 | 当前天气、小时级趋势、未来 3 至 5 天预报 |

#### 配置项

- 城市或定位标识
- 温度单位，默认摄氏度
- 刷新频率，默认 30 分钟
- 是否显示小时级预报
- 是否显示未来几天预报

#### 交互要求

- 支持手动刷新
- 支持缓存最近一次成功结果
- 失败时保留卡片结构并展示错误提示

## 9. 创建弹窗需求

### 9.1 布局

- 左侧：组件类型列表或分类卡片
- 中部：尺寸预设列表
- 右侧：配置面板 + 实时预览

### 9.2 交互

- 切换类型时重置为该类型默认尺寸与默认配置
- 切换尺寸时保留可复用配置项
- 确认按钮只有在必填配置完成后可点击

### 9.3 快捷卡片兼容

- 现有通用组件能力保留为“快捷卡片”
- 快捷卡片仍允许配置图标、名称、打开动作、背景
- 快捷卡片的尺寸策略可保持现状，但建议同样隐藏 1x1

---

## 10. 数据模型需求

建议在现有首页 widget 模型上增加：

- `sourceType`: `builtin | shortcut | plugin`
- `widgetType`: `pomodoro | date | weather | shortcut`
- `sizePreset`: 如 `2x2`、`4x2`、`4x3`
- `widgetConfig`: JSON 字符串或对象

约束：

- `sourceType=builtin` 时，`widgetType` 必填
- `sizePreset` 必须属于该 `widgetType` 的允许集合
- `widgetConfig` 结构按 `widgetType` 校验

---

## 11. 非功能需求

### 11.1 性能

- 首页初次渲染不因单个天气组件请求而阻塞整体布局
- 单个页面存在 20 个以上组件时，拖拽和切换分类仍需保持流畅
- 番茄钟组件同时存在多个实例时，不应产生明显卡顿

### 11.2 可维护性

- 新增内置组件时，应只需新增注册定义、配置表单、渲染组件
- 不应在 `GridItem.vue` 内继续堆叠 `if/else` 业务分支

### 11.3 可扩展性

- 为后续插件组件接入预留类型入口
- 为更多首页组件类型预留 schema 扩展空间

### 11.4 国际化

- 组件名称、状态文本、天气状态文案进入 i18n 体系

## 12. 验收标准

### AC-HW-01

- 首页创建组件时，可以直接看到“番茄钟 / 日期 / 天气 / 快捷卡片”

### AC-HW-02

- 所有内置组件都无法创建 1x1 尺寸

### AC-HW-03

- 番茄钟、日期、天气均至少支持 2 个以上尺寸，且不同尺寸布局明显不同

### AC-HW-04

- 创建和编辑时都可以修改尺寸与业务配置

### AC-HW-05

- 旧组件数据加载正常，不会因为缺失新字段而报错

### AC-HW-06

- 天气组件在未配置、加载失败、有缓存三种情况下都有明确显示

### AC-HW-07

- 番茄钟在切换首页分类、切换路由、窗口失焦后返回，剩余时间仍正确

---

## 13. 建议首期交付边界

为了控制复杂度，建议首期按以下标准交付：

- 必做：
  - 内置组件创建流程
  - 番茄钟 3 个尺寸
  - 日期 3 个尺寸
  - 天气 2 至 3 个尺寸
  - 数据模型扩展
  - 旧数据兼容

- 可延后：
  - 日期农历/节假日
  - 天气更多指标
  - 插件组件化接入
  - 番茄钟跨重启恢复
