# 知识库功能需求

> 版本：0.1
> 日期：2026-05-27
> 文档状态：草案

## 1. 目标

知识库是 GuYanTools 桌面端的本地资料中枢，服务于文件整理、笔记创作、速记捕获、Todo 联动和未来 AI 上下文检索。它不是单独的笔记应用，而是现有工具集内的资料组织层。

## 2. 用户对象

- 开发者：保存项目资料、命令、脚本说明、接口文档、排障记录。
- 学习/研究用户：保存 PDF、Word、PPT、Excel、Markdown、网页摘录和阅读笔记。
- 任务管理用户：把临时便签、会议记录、资料摘录转为 Todo。
- AI 用户：后续希望对本地资料做摘要、问答、整理和任务生成。

## 3. 核心场景

### 3.1 本地资料入库

用户可以把 Markdown、PDF、Word、PPTX、Excel、图片和普通文本导入知识库。首版保留原文件，生成元数据和索引文本，后续提供预览、摘录和缩略图。

### 3.2 Markdown 创作

用户可以新建 Markdown 页面，编辑和保存内容。编辑器长期目标是接近 Typora：源码模式、Live Preview、目录、图片粘贴、Focus Mode、Typewriter Mode 和导出。

### 3.3 Sticky Notes 速记

用户可以通过快捷键唤出速记窗口，快速记录内容。速记默认进入快速收集箱，可搜索、打标签、归档，并可转成知识库页面或 Todo。

### 3.4 OneNote-like 笔记

用户可以创建更自由的块笔记页面，支持文本、任务、图片、附件、链接和后续画布内容。GuYanTools 不兼容 `.one` 文件，而是提供自有 `.gynote` 便携格式。

### 3.5 Todo 联动

页面内任务块、速记和摘录都可以转为 Todo。Todo 需要保留来源页面、块或便签的反链，便于回溯上下文。

### 3.6 AI 预留

知识库需要预留 AI 分块、embedding、引用来源和操作日志。AI 默认关闭，必须由用户配置 Provider 后主动触发。

## 4. 功能需求

### 4.1 信息架构

- 支持 Library：知识库顶层边界。
- 支持 Space：项目或主题空间。
- 支持 Folder：树形文件夹。
- 支持 Page：Markdown、块页面、画布页面、外部文档页面。
- 支持 Attachment：原始文件、图片、截图、预览缓存。
- 支持 Tag：跨文件夹分类。
- 支持 Link：页面、块、附件、Todo、外链之间的引用关系。

## 5. 非功能需求

- 本地优先：文件和索引默认存储在本地。
- 安全：renderer 不直接访问 Node、数据库或文件系统。
- 性能：导入、抽取、索引、预览生成必须后台化。
- 可恢复：页面更新和节点移动使用事务；删除优先软删除或归档。
- 稳定性：导入前必须有大文件保护；抽取失败必须可见且可重试；索引任务必须可取消或进入明确终态。
- 配置：桌面端必须提供知识库配置入口，覆盖默认库、附件目录策略、LibreOffice 路径、索引开关、单文件导入上限和预览缓存清理。
- 可扩展：页面类型、附件类型、索引任务和 AI 分块结构要可扩展。
- 可验证：Rust service 需要单元测试；桌面端需要构建验证。

## 6. 非目标

- 不兼容 Microsoft OneNote `.one`。
- 不实现 Office 原格式编辑。
- 不首版打包 LibreOffice、Tika 或向量数据库。
- 不首版做多人协作、账号同步和冲突合并。
- 不默认启用 AI、embedding 或自动总结。
