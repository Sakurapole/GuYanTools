# GuYanTools AI Agent / Chat 功能需求文档

> **版本**：1.0  
> **日期**：2026-03-19  
> **文档状态**：草案

---

## 1. 项目背景

GuYanTools 是一个基于 Electron 37 + Vue 3 + TypeScript + Pinia 的桌面工具集应用。当前需要集成 AI 能力，支持两种核心模式：

- **Chat 模式**：通用 AI 对话，类似 CherryStudio，支持多提供商、多模型
- **Agent 模式**：AI Agent 能力，分为 Code Agent 和 General Agent 两个大类

---

## 2. 核心概念定义

### 2.1 Chat 模式 vs Agent 模式

| 维度         | Chat 模式                  | Agent 模式                       |
| ------------ | -------------------------- | -------------------------------- |
| **交互模式** | 单轮/多轮纯文本对话        | 任务驱动，AI 自主执行操作        |
| **工具执行** | 无                         | 有（自动执行文件操作、命令等）   |
| **UI 风格**  | 传统聊天气泡               | 含代码块、终端输出、文件 diff 等 |
| **上下文**   | 用户手动输入               | 自动感知项目/环境                |
| **用途**     | 问答、翻译、写作、知识检索 | 代码生成、自动化任务、数据分析   |

### 2.2 Agent 模式的两个大类

#### Code Agent（Codex Agent）
- **实现方式**：核心逻辑通过 `@openai/codex-sdk` 实现
- **特点**：高度自定义 UI 样式，类终端/IDE 风格
- **能力**：代码理解、代码生成、代码修改、Bug 修复、命令执行
- **应用层仅负责**：UI 渲染和事件转发（核心逻辑全由 SDK 处理）

#### General Agent（自定义 Agent）
- **实现方式**：基于 OpenAI / Claude / Ollama 等 API + 自定义 Tool 系统
- **特点**：灵活的 Agent 配置，可自定义系统提示词和工具集
- **能力**：取决于配置的工具集（文件操作、Web 搜索、数据分析等）
- **应用层负责**：Provider 适配、工具执行、上下文管理、流式处理

## 3. 功能需求

### 3.1 Chat 模式需求

#### FR-CHAT-01：多提供商管理
- 支持添加/编辑/删除 AI 提供商
- 支持的提供商类型：OpenAI、Claude、Ollama（本地模型）、自定义兼容 API
- 每个提供商需配置：名称、API Base URL、API Key、支持的模型列表
- API Key 仅存储在主进程，不暴露给渲染进程

#### FR-CHAT-02：模型管理
- 每个提供商下可配置多个模型
- 模型属性：名称、最大上下文长度、是否支持流式、是否支持视觉
- 支持设置默认模型
- 支持运行时切换模型

#### FR-CHAT-03：会话管理
- 支持创建/删除/重命名会话
- 会话列表按时间排序，支持置顶
- 支持会话搜索
- 会话数据持久化至本地数据库

#### FR-CHAT-04：消息交互
- 支持发送纯文本消息
- 支持流式响应（SSE），逐字显示 AI 回复
- 支持停止当前生成
- 支持重新生成回复
- 支持 Markdown 渲染（含代码高亮、数学公式）
- 支持复制消息内容

#### FR-CHAT-05：上下文管理
- 支持系统提示词配置
- 支持上下文窗口设置（最大历史消息数）
- 支持温度、最大输出 Token 等参数调节

---

### 3.2 Agent 模式需求 — Code Agent（Codex）

#### FR-CODEX-01：Codex 环境管理
- 启动前检测 Codex CLI 是否可用
- 支持配置 OPENAI_API_KEY
- 支持配置自定义 API Base URL
- Codex 实例生命周期管理（懒初始化，按需创建/销毁）

#### FR-CODEX-02：项目/工作目录管理
- 支持选择本地项目目录作为工作目录
- 提示是否为 Git 仓库（Codex 默认要求）
- 提供跳过 Git 仓库校验的选项

#### FR-CODEX-03：代码 Agent 会话
- 基于 `Thread` 模型管理会话
- 支持多轮对话（同一 Thread 内 `run()` 多次）
- 支持恢复已有线程（`resumeThread()`）
- 展示完整的执行过程（工具调用、文件操作、命令执行）

#### FR-CODEX-04：流式事件展示
- 使用 `runStreamed()` 获取流式事件
- 实时展示：
  - 文本输出（推理过程）
  - 工具调用过程（执行了什么操作）
  - 文件变更（修改了哪些文件）
  - 命令执行（执行了什么终端命令及输出）
- `turn.completed` 后展示 usage 统计

#### FR-CODEX-05：高度自定义 UI
- 类终端/IDE 风格，非传统聊天气泡
- 代码块使用等宽字体 + 语法高亮
- 文件变更以 diff 形式展示
- 终端输出以终端模拟器风格展示
- 工具调用以可折叠卡片展示执行详情

#### FR-CODEX-06：结构化输出支持
- 支持为 Codex 请求配置 JSON Schema 输出格式
- 结构化结果以格式化方式展示

#### FR-CODEX-07：图片输入支持
- 支持附加本地图片（截图、设计稿等）
- 结合文本提示一起发送

---

### 3.3 Agent 模式需求 — General Agent（自定义 Agent）

#### FR-AGENT-01：Agent 配置管理
- 支持创建/编辑/删除自定义 Agent
- Agent 属性：名称、系统提示词、绑定的提供商和模型、温度、最大输出 Token、启用的工具列表
- 支持预置 Agent 模板

#### FR-AGENT-02：工具系统
- 内置工具：文件读取、文件写入、命令执行
- 支持工具的启用/禁用
- 工具执行结果自动回传给模型
- 工具调用过程在 UI 中可视化

#### FR-AGENT-03：Agent 会话
- 基于 Conversation 模型管理会话
- 支持多轮对话，自动携带上下文
- 工具调用循环：模型请求工具 → 执行工具 → 返回结果 → 模型继续

#### FR-AGENT-04：流式响应
- 使用 Provider 适配器的 `streamChat()` 获取流式响应
- 逐字展示 AI 回复
- 实时展示工具调用过程

---

### 3.4 通用需求

#### FR-COMMON-01：模式切换
- 顶部或侧边栏提供 Chat / Agent 模式切换入口
- Agent 模式内进一步提供 Code Agent / General Agent 的切换
- 切换模式时，各模式的会话状态独立保持

#### FR-COMMON-02：设置页面
- 在现有 Settings.vue 中增加 AI Agent 配置面板
- 提供商管理 → 模型管理 → Agent 配置的分级管理
- Codex Agent 的独立配置区域

#### FR-COMMON-03：数据持久化
- 会话数据存储在本地 SQLite 数据库
- 配置数据存储在应用配置 JSON（`features.aiAgent`）
- 支持数据导出/导入

#### FR-COMMON-04：错误处理
- 网络错误时显示友好提示，支持重试
- API Key 无效时引导用户到设置页面
- SSE 流中断时可恢复
- 工具执行失败时不阻塞对话

## 4. 非功能需求

### 4.1 性能

| 指标               | 目标                        |
| ------------------ | --------------------------- |
| 消息发送到首次响应 | < 500ms                     |
| 流式响应帧率       | 无明显卡顿，≥ 30fps UI 更新 |
| 会话列表加载       | < 200ms                     |
| 千条消息历史加载   | < 500ms                     |
| Codex 线程启动     | < 2s                        |

### 4.2 安全

- API Key 仅在主进程持有，不通过 IPC 暴露
- 工具执行（write_file、execute_command）需安全策略控制
- Codex 沙箱内执行，遵循其内置安全机制

### 4.3 可维护性

- 模块化设计，Provider / Agent / Tool 解耦
- 遵循现有插件架构 L0-L5 分层
- TypeScript 类型安全

### 4.4 兼容性

- 支持 Windows 10+、macOS 12+、Ubuntu 22.04+
- Codex CLI 需单独安装（仅 Code Agent 模式需要）

## 5. 用户故事

### US-01：Chat 模式基本对话
**作为** 用户，**我想** 在聊天界面中与 AI 对话，**以便** 获取问题解答和文本生成帮助。

### US-02：配置 AI 提供商
**作为** 用户，**我想** 添加自己的 OpenAI API Key 和自定义 API 地址，**以便** 使用我自己的 AI 服务。

### US-03：Code Agent 代码修复
**作为** 开发者用户，**我想** 选择一个本地项目目录，让 Code Agent 分析代码并修复 Bug，**以便** 提高开发效率。

### US-04：General Agent 自定义任务
**作为** 用户，**我想** 创建一个自定义 Agent（指定系统提示词和工具），**以便** 完成特定领域的自动化任务。

### US-05：模式切换
**作为** 用户，**我想** 在 Chat 和 Agent 模式间自由切换，**以便** 根据不同场景选择最合适的交互方式。

## 6. 验收标准

### 6.1 Chat 模式
- [ ] 能添加 OpenAI 提供商并完成连接测试
- [ ] 能创建会话并发送消息，收到流式回复
- [ ] 能切换模型并正常对话
- [ ] 消息支持 Markdown 渲染和代码高亮
- [ ] 能停止生成中的回复

### 6.2 Code Agent 模式
- [ ] 能选择项目目录并启动 Codex 线程
- [ ] 能发送提示并看到流式执行过程
- [ ] 能看到工具调用和文件变更详情
- [ ] 支持多轮对话
- [ ] 线程信息正确持久化和恢复

### 6.3 General Agent 模式
- [ ] 能创建自定义 Agent（含系统提示词和工具配置）
- [ ] Agent 能执行工具调用并展示执行过程
- [ ] 工具执行结果能正确回传给模型

### 6.4 通用
- [ ] 模式切换流畅，状态互不干扰
- [ ] 设置页面完整可用
- [ ] 网络异常时有友好提示

## 7. 术语表

| 术语              | 定义                                           |
| ----------------- | ---------------------------------------------- |
| **Provider**      | AI 服务提供商（如 OpenAI、Claude、Ollama）     |
| **Chat 模式**     | 纯对话模式，无工具执行能力                     |
| **Agent 模式**    | AI 可自主执行操作的模式                        |
| **Code Agent**    | 基于 `@openai/codex-sdk` 的代码 Agent          |
| **General Agent** | 基于标准 API + 自定义工具的通用 Agent          |
| **Thread**        | Codex SDK 中的线程/会话概念                    |
| **Conversation**  | 自定义 Agent/Chat 中的会话概念                 |
| **Tool**          | Agent 可调用的外部工具（文件读写、命令执行等） |
