# 同步中心使用说明

本文说明如何在 GuYanTools 中使用同步中心同步应用配置、知识库和 AI 元数据。同步中心入口位于：

```text
设置 -> 同步中心
```

## 同步内容

当前同步中心支持以下内容：

| 类型 | 同步内容 | 说明 |
| --- | --- | --- |
| 应用配置 | 外观、底栏、快捷键、功能配置、配置档案 | 每份配置都有唯一 profileId 和设备 ID。 |
| 知识库 | 库、空间、文件夹、页面、附件元数据、标签、链接 | 身份以 ID 为准，名称只用于展示和冲突识别。 |
| AI | 助手、服务商、模型配置等元数据 | 不同步 API Key、访问令牌、MCP 环境密钥。 |
| 附件 | 知识库附件文件 | 通过内容哈希上传和拉取。 |

不会同步的内容：

- AI API Key、服务商密钥、MCP token、password、secret 等敏感值。
- SSH / FTP 密码、私钥口令、插件密钥。
- 明确属于本机路径或本机环境的运行缓存。

## 推荐使用方式

个人多设备同步优先使用 WebDAV。当前内置坚果云 WebDAV 预设，也支持自定义 WebDAV。

多设备或长期公网访问可以使用自建同步后端。自建后端支持账号、设备绑定、服务端修订号、冲突记录和附件存储，更适合固定服务器部署。

## 使用坚果云 WebDAV

### 1. 准备坚果云应用密码

在坚果云网页端创建第三方应用密码。不要使用坚果云登录密码。

需要准备：

```text
WebDAV 地址：https://dav.jianguoyun.com/dav/
账号：坚果云账号邮箱
第三方应用密码：坚果云生成的应用密码
远程目录：GuYanTools/Sync
```

### 2. 在应用内配置

进入：

```text
设置 -> 同步中心 -> WebDAV
```

填写：

```text
服务类型：坚果云 WebDAV
WebDAV 地址：https://dav.jianguoyun.com/dav/
账号：你的坚果云账号邮箱
第三方应用密码：坚果云应用密码
远程目录：GuYanTools/Sync
```

然后依次点击：

```text
保存配置
测试连接
立即同步
```

密码只保存在本机安全存储中，不会写入同步配置。

### 3. 第二台设备接入

在第二台设备填写相同的 WebDAV 配置，然后点击：

```text
测试连接
立即同步
```

同步完成后，在“配置档案”区域可以看到来自其他设备的配置。可以选择“应用”或“设为默认”。

## 使用自建同步后端

### 1. 部署后端

后端部署说明见：

```text
sync_server/README.md
```

推荐公网部署只开放：

```text
80/tcp
443/tcp
```

PostgreSQL、Redis、MinIO 和后端裸端口默认不要直接暴露到公网。推荐通过 Caddy 或 Nginx 把 HTTPS 域名反代到本机 `127.0.0.1:38420`。

应用内服务地址推荐填写：

```text
https://sync.example.com
```

局域网或临时测试可以填写：

```text
http://<server-ip>:38420
```

### 2. 在应用内配置

进入：

```text
设置 -> 同步中心 -> 自建同步后端
```

填写：

```text
服务地址：https://sync.example.com
账号邮箱：你的同步账号邮箱
账号密码：同步账号密码
设备名称：用于识别当前设备的名称
```

然后点击：

```text
登录并绑定当前设备
```

应用会自动完成：

1. 登录账号。
2. 必要时自动注册账号。
3. 注册当前设备。
4. 保存 access token、refresh token 和 device token。

不需要手动填写访问令牌或设备令牌。同步中心只显示“令牌已保存”状态，不会把令牌明文展示在界面中。

登录绑定后可以点击：

```text
测试连接
立即同步
```

### 3. 退出和解绑

如果不再使用当前设备同步，点击：

```text
退出并解绑当前设备
```

应用会清除本机保存的同步后端令牌，并尝试在后端撤销当前设备。

## 配置档案

同步中心会展示本机和远端的配置档案：

| 操作 | 作用 |
| --- | --- |
| 应用 | 立即把该配置应用到当前应用。 |
| 设为默认 | 以后默认使用该配置，但不会立即应用。 |
| 当前 | 表示当前正在使用的配置。 |
| 默认 | 表示启动时默认使用的配置。 |

第一次同步时，如果远端没有配置，当前设备会上传本机配置。其他设备同步后可以拉取并选择使用哪一份配置。

## 冲突处理

当两个设备在同一个基础版本上修改了同一个对象，就会产生冲突。冲突会显示在“冲突”区域。

可选处理方式：

| 操作 | 作用 |
| --- | --- |
| 使用本机 | 保留当前设备的版本，并重新上传。 |
| 使用远端 | 使用远端版本覆盖本机对应对象。 |
| 保留两份 | 两份都保留。应用配置会保存为额外配置档案；知识库同名不同 ID 会保留为两个对象。 |

处理冲突后，再点击“立即同步”，把处理结果上传到远端。

## 自动同步

应用启动后会自动尝试同步。运行中也会按固定间隔自动同步；本机配置变更后会做短时间防抖，再触发同步。

如果网络失败或后端不可用，同步器会进入退避等待，不会持续高频重试。恢复网络后可以点击“立即同步”手动触发。

## 常见问题

### WebDAV 测试连接失败

检查：

1. WebDAV 地址是否正确。
2. 坚果云是否使用了第三方应用密码，而不是登录密码。
3. 账号邮箱是否正确。
4. 远程目录是否有写入权限。

坚果云推荐地址：

```text
https://dav.jianguoyun.com/dav/
```

### 自建后端测试连接失败

在服务器上先检查：

```bash
curl --noproxy '*' -fsS http://127.0.0.1:38420/readyz
systemctl status guyantools-sync-server
journalctl -u guyantools-sync-server --no-pager -n 120
```

如果使用域名和 HTTPS，再检查：

```bash
curl -fsS https://sync.example.com/readyz
```

应用内不要填写 PostgreSQL、Redis、MinIO 的端口。只填写同步后端 HTTP API 地址或 HTTPS 域名。

### 为什么第二台设备没有 AI API Key

这是预期行为。同步中心只同步 AI 服务商、助手和模型元数据，不同步 API Key。第二台设备需要在 AI 设置中重新填写自己的 API Key。

### 同名知识库会不会覆盖

不会。知识库身份以 ID 为准。名称相同但 ID 不同的库、空间、页面会被视为不同对象，冲突处理时也可以选择保留两份。

### 能不能同时配置 WebDAV 和自建后端

可以保存两类配置，但当前同步运行时只使用当前选中的 provider。保存 WebDAV 配置会切换到 WebDAV；保存或登录自建后端会切换到自建后端。

### 什么时候需要手动点击立即同步

以下情况建议手动点击：

1. 第一次配置同步源后。
2. 第二台设备刚接入后。
3. 处理冲突后。
4. 网络恢复后想立即重试。
